작업 마무리 점검을 수행합니다. 아래 3단계를 병렬 서브에이전트로 실행한 뒤 결과를 취합해 주세요.

## 1단계: 코드 리팩토링 점검 (서브에이전트 1)

변경된 파일을 중심으로 전체 코드를 점검합니다:
- `git diff HEAD~1 --name-only`로 최근 변경 파일 파악
- 변경된 파일 + 관련 파일 읽고 점검:
  - 타입 에러, 미사용 import/변수
  - 보안 취약점 (인증 누락, 입력 검증 등)
  - 에러 핸들링 누락
  - 코드 중복
  - 네이밍 일관성
- **실제 이슈만** 보고 (스타일 취향 제외)

## 2단계: 컨텍스트 문서 현행화 (서브에이전트 2)

CLAUDE.md와 MEMORY.md를 현재 코드베이스와 비교합니다:
- `CLAUDE.md` 읽고 현재 코드와 불일치하는 부분 찾기:
  - 새로 추가된 파일/라우트/기능이 문서에 빠져있는지
  - DB 스키마 변경 반영 여부
  - 환경변수, 외부 서비스 정보 최신 여부
  - 삭제/변경된 내용이 문서에 남아있는지
- MEMORY.md도 동일하게 점검

## 3단계: 결과 취합 및 적용

서브에이전트 결과를 종합하여:
1. **코드 이슈가 있으면** → 수정 적용 후 사용자에게 변경 사항 보고
2. **문서 업데이트가 필요하면** → CLAUDE.md / MEMORY.md 수정
3. **모두 정상이면** → "점검 완료, 이슈 없음" 보고
4. 사용자에게 커밋/푸시 여부 확인
