# SL Steel 홈페이지 구조 & 섹션별 핵심 메시지

## 기술 스택
- Next.js 16 (App Router)
- React 19, TypeScript 5
- Tailwind CSS v4
- 배포 예정: Vercel

---

## 전체 URL 구조

```
/          랜딩 페이지 (단일 스크롤 페이지)
/blog      블로그 목록
/blog/[slug]  블로그 포스트
```

---

## 랜딩 페이지 섹션 순서 및 핵심 메시지

```
Hero → Stats → About → Products → Business → Process → System → Why → Contact
```

### Hero
**메인 카피**: "공사 후 남은 철근, 최고가로 매입합니다"
**서브카피**: 녹이 슬었거나 절단된 철근도 명확한 기준으로 현장에서 직접 수거
**CTA**: 매입 문의 / 납품 문의
**배경**: 다크 네이비 + 현장 이미지 캐러셀 (3장)

### Stats (신뢰 지표)
- 2009년~ (업력)
- 중부·남부권 (전국 거점 네트워크)
- 4m↑ (절단 철근 매입 기준)
- 당일 수거 (현장 직접 방문)

### About
**제목**: "공사 후 현장에 남은 철근, SL Steel이 가져갑니다"
**고객 여정 3단계**: 전화 한 통 → 현장 직방문 → 즉시 정산

### Products (매입 기준)
| 품목 | 기준 | 매입 여부 |
|------|------|-----------|
| 녹 발생 철근 | 온전한 길이 | ✓ 가능 |
| 절단 철근 | 4m 이상 | ✓ 가능 |
| 매입 불가 항목 | 휘거나 3.5m 이하·고철 | ✗ 불가 |

### Business
- **매입 (주력)**: 현장 직접 수거, 빠른 정산, 보관 공간 확보, 합리적 매입가
- **납품 (보조)**: 대량 재고 보유, 현장 직납, 긴급 대응, 안정 공급

### Process
**매입 프로세스**: 문의 → 상태 확인 → 매입가 제시 → 현장 수거 → 정산
**납품 프로세스**: 문의 → 견적/일정 → 출고 → 현장 납품 → 사후 대응

### System (자체 인프라)
- 대형 창고: 수거 즉시 입고·보관
- 야적장: 대형 물량도 안정 대응
- 크레인 카고 트럭: 소형·대형 물량 모두 대응

### Why (선택 이유)
1. 녹슨 철근·4m↑ 절단 철근 모두 명확한 기준으로 매입
2. 5톤·25톤 크레인 카고 트럭으로 현장 직수거 — 별도 운반 불필요
3. 최고가 매입 — 상품성 없어도 최고가에 가져갑니다
4. 매입과 납품을 함께 운영하는 현장 밀착 구조

## 블로그 (`/blog`)

랜딩 페이지와 완전히 분리된 독립 섹션. 별도 헤더(BlogNav)와 레이아웃.

**구조**
- 목록 페이지: 카테고리 탭 필터 + 사이드바 (CTA·카테고리·최근글·키워드)
- 포스트 페이지: 본문 + 사이드바 + 포스트 하단 CTA

**카테고리**
매입 기준 / 현장 실무 / 업계 정보 / 시설·인프라 / 수거 사례

**콘텐츠 관리**
현재: MDX 파일 (`content/blog/*.mdx`)
향후: Supabase DB로 마이그레이션 예정
